# 09 效果驱动的消息类型与回复类型设计

> 这是一个设计文档，不给具体回复内容，只说明如何根据不同目标效果，从素材中提取消息类型与回复类型。

## 1. 目标

本设计旨在将 `sucai.md` 的素材风格，转化为一个可复用的“效果驱动型 Skill 结构”：

- 不同目标效果对应不同回复策略
- 不同消息类型映射到不同风格分类
- 以素材中出现的类型为标签库，而不是具体句子
- 支持按角色对话场景选择最合适的类别

## 2. 关键维度

### 2.1 目标效果（Effect）

1. 维持关系（Maintain）
2. 升级关系（Upgrade）
3. 建立边界（Boundary）
4. 化解尴尬 / 解除冲突（Defuse）
5. 制造互动 / 提升氛围（Engage）

### 2.2 消息类型（Message Type）

1. 自我贬低 / 负面情绪
2. 状态通知 / 日常动作
3. 夸奖 / 正面反馈
4. 提问 / 试探
5. 冲突 / 攻击
6. 暧昧 / 撩拨
7. 关心 / 疼惜
8. 道歉 / 折中

### 2.3 风格标签（Style Tags）

从 `sucai.md` 中提取的标签作为设计原料：

- 心疼型、夸奖型、真想谈型、玩梗型、霸总型、反问型、调侃型、互动型、走心型、直球型、反向型、讽刺型、环保型、把风型、祝福型
- 这些标签不是句子，而是“策略节点”

## 3. 设计框架

### 3.1 效果→主策略映射

| 目标效果 | 最佳策略类别 | 说明 |
|----------|-------------|------|
| 维持关系 | 心疼型 / 夸奖型 / 走心型 / 祝福型 | 给对方安全感、陪伴感、善意感 |
| 升级关系 | 真想谈型 / 撩拨型 / 反问型 / 互动型 | 使对话向更深层次推进 |
| 建立边界 | 霸总型 / 直球型 / 打断型 / 反问型 | 表明立场、保持尊重和距离 |
| 化解冲突 | 脱身型 / 自嘲型 / 讽刺型 / 反向型 | 转移焦点、缓和紧张、保全关系 |
| 提升氛围 | 玩梗型 / 互动型 / 娱乐型 / 可爱型 | 让聊天变得更轻松、更好玩 |

### 3.2 消息类型→策略选择

对于每类消息，优先匹配最能达成目标的策略：

- 自我贬低：优先心疼型 → 夸奖型 → 真想谈型
- 状态通知：优先反问型 → 互动型 → 撩拨型
- 夸奖：优先真想谈型 → 反问型 → 自恋/玩梗型
- 提问/试探：优先反问型 → 霸总型 → 直球型
- 冲突/攻击：优先霸总型 → 自嘲型 → 讽刺型
- 暧昧/撩拨：优先撩拨型 → 互动型 → 真想谈型
- 关心/疼惜：优先走心型 → 祝福型 → 夸奖型
- 道歉/折中：优先高情商型 → 反问型 → 走心型

### 3.3 角色与效果的关系

角色决定同一策略的语气和强度：

- crush：策略可偏撩和温柔
- 朋友：策略可偏幽默和互动
- 同事：策略可偏稳重和礼貌
- 上级：策略可偏简洁和尊重
- 恋人：策略可偏深情和承诺
- 普通熟人：策略可偏轻松和不过界

## 4. 从素材到设计的转换

### 4.1 以类型标签替代具体句子

`sucai.md` 的每一个句式都属于一种策略标签。

例如：
- “你在质疑我的眼光吗？” → 真想谈型 + 反问型
- “我是问你醒了没？” → 调侃型 + 互动型
- “个人情绪不用对我汇报。” → 霸总型 + 界定边界

### 4.2 设计流程

1. 识别对方消息类型（上述 8 类）
2. 明确目标效果（5 种之一）
3. 从 `sucai.md` 标签库中选 2-3 种策略标签
4. 按角色调整语气和表达强度
5. 生成可选类型列表，供用户挑选

## 5. 设计示例结构

一个 Skill 的“输出结构”应包含：

- 目标效果说明
- 当前消息类型
- 适合的回复类型列表
- 每种类型的简要逻辑说明
- 可选的首推策略

例如：

```
目标：升级关系
消息类型：状态通知
适合策略：反问型、互动型、撩拨型
说明：对方通过日常动作试探你的关注度，最适合用反问或轻撩把话题拉回你们两个人。
```

## 6. 设计落地建议

### 6.1 在 `/references` 里维护两个层次

- 研究层：`references/research/09-effect-message-design.md`，描述设计原则和映射机制
- 生成层：`references/prompt.md` / `references/SKILL.md`，用于实际调用和输出

### 6.2 让 Skill 输出“类型供选择”而不是“具体回复"

Skill 应该直接返回：

- 心疼型、玩梗型、真想谈型、霸总型、反问型
- 每个类型配一个简要说明
- 让用户按效果和角色选择，而不直接给最终句子

### 6.3 未来扩展

- 可以把 `sucai.md` 的标签库进一步整理成“类型词典”
- 可以为每个标签准备“语气权重”和“角色权重”
- 让 Skill 自动推荐 2-3 个最匹配的策略组合

## 7. 本设计的价值

- 保持和素材一致的风格，但避免机械复刻句子
- 从“效果”出发，适配更多变体和不同对话场景
- 支持你按“目标选择类型”，而不是按“具体回复复制”
- 更符合你这次需求：你要的是素材分析设计，而不是具体句子
